    Gitlet is a mini version control system that mimics the functionality of the version control system Git. Gitlet uses SHA1 hashcodes, file persistence, serialization, various data structures, and graph traversals to store and track the contents of files and all of their past versions. Gitlet supports many of Git's common commands, such as init, add, commit, status, log, branch, merge, checkout, reset, and remove.

    SIXT33N is a voice-controlled robot car. The front-end of the car consists of a mic-board connected to filtering circuit. After filtering the signal from the mic-board with a high pass filter, the signal is sent to a TI Launchpad. The Launchpad uses Principal Component Analysis and Singular Value Decomposition to classify the signal. If the signal matches one of the preset commands, the car will drive as instructed (straight, left, or right). Closed-loop feedback and linearized models are also used to ensure that the car drives straight.

    SIXT33N Robot Car

    TI Launchpad, Circuits, Python, C

    This project is a simulation for a popular game, Connect Four. The game is presented in an interactive manner with GUI displays and buttons. The game supports both single player vs. AI and a multiplayer option. The AI computer player uses a strategy based on the minimax algorithm. The AI computer player also uses a heuristic, consisting of numeric calculations that quantify a board's potential for both offense and defense, to select its next move.

    Lines of Action (LOA) is a checker like board game invented by Claude Soucie. My implementation of LOA features GUI components and an AI player. The program uses game trees, the minimax algorithm, and alpha-beta pruning to select the best move for the AI Computer Player. My implementation also features a heuristic function that evaluates the strength of a given board state for either player based on group sizes and proximity.

    Enigma is a model of the Enigma Machines used by Germany in World War II for military communications. This program uses object-oriented programming to model the various electrical and mechanical components of a real Enigma machine: rotors, reflectors, pawls, notches, plugboard, and notches. This program parses a configuration file with Regex to set up an Enigma Machine model with the specified rotors, reflectors, notches, permutations, alphabet, and plugboard connections. Then, it scans an input text file and writes the corresponding encryption into an output file.
